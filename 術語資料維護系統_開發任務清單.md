# 術語資料維護系統 - 開發任務清單

## 一、需求確認與設計
- 完成 PRD 確認會議紀錄
- 欄位字典與驗證規則最終定版
- UI Wireframe 版面確認
- API 規格與錯誤碼對齊

## 二、後端開發
- 資料表建立與遷移（Term、TermLanguage、TermAlias、Category、TermVersion）
- 基礎 CRUD API（術語、分類）
- 動態語言支援（語言欄位儲存與查詢）
- 搜尋/過濾與排序 API
- 批次作業 API
- CSV 匯入/匯出 API（含預覽）
- 版本紀錄與差異比對 API
- 錯誤碼與回應格式統一

## 三、前端開發
- 列表頁（搜尋、過濾、排序）
- 欄位顯示自訂與固定欄位
- 新增/編輯表單（動態語言欄、別名、備註）
- 批次操作條與確認視窗
- CSV 匯入流程（映射、預覽、結果）
- CSV 匯出
- 版本紀錄與差異比對頁
- 空狀態與錯誤提示

## 四、測試與品質
- API 單元測試與整合測試
- 匯入/匯出測試（含錯誤案例）
- 多語缺值搜尋測試
- 批次操作回歸測試
- 版本差異比對測試

## 五、驗收與上線準備
- 驗收清單對照
- Demo 與回饋修正
- 上線說明與操作文件
